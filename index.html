Overview

Have you ever had a canceled flight? Have you ever been stuck at an airport because of delays and you don't know where to go ? Srsly is here to help. The APP helps travelers find a convenient hotel and gives them the option to book a direct uber ride to it as well . Srsly is my first group project during the Northestern web developement Bootcamp.

Description

The landing page presents the App's mission,the project's information and a search bar. The search is by airport, and once the airport selected, the App will take you to the seconde page where you can see a map with the your position and the nearby hotels located on it.
the page will also show the list of hotels with their pictures and all kind of infomation such as the price, ratings, amenities and rooms availability.
Clicking on the car icon would take you to the third page where you can book a ride through the Uber API. the App would automatically calculate the fare and prompt you to confirm the ride.

Team Roles

In the Srsly project I worked on implementing the token based authentication (Oauth) to authenticate and make calls to the Amadeus API (The API provider we used for Srsly).
I also worked on the google maps API to translate the coordinations of the hotels I get from amadeus into a map where you can see all the available hotels

Overview

The idea behind this App is to allow the users to search the extensive google books API in simplified and fast way. it also allows you to save the searched books to the database. this app was built using the MERN technology stack

Description

The utilization of this App is very easy. Once prompted the the page you will enter the search term in the search bar located right below the naigation bar. the App is designed to interact with every keyboard press as it keeps updating the search results with every change in the search bar.
Every book resulting from the search would be represented in a card with its cover, title, description, link to the book and a button to save the book in the database. To see the saved books click on the "saved books" link in the navigation bar.

Overview

Description

Overview

For the Last group project of my Northestern coding boostcamp my team had the idea of building a social media application to help people create events and activities and use them as platforms to invite their friends, share information about the event, share videos and images, and keep track of updates. Whether it's a wedding, a birthday party or a meetup, the application gives the user a lot of flexibility in creating events and gives them the option of making them public or private.

Description

Eventbook's main features are :
1- Signing using Facebook and Google: using Passeport oauth to authenticate the users through the Facebook and Google APIs
2- Creating Events and managing them : creating the events and setting their parameters (evet name, event description, event date, event type, event banner ...etc)
3- Searching and inviting users
4- Searching public events and private invitations
4- live commenting and multimedia sharing through the event platform
6- Profile management page

Team Roles

In the EventBook project I assumed the role of project manager and helped ensure that the application development process works as it is intended. Through this role, I was able to organize the team's communication,set goals and objectives, assign individual tasks for each member, and make sur we meet the project requirement as we progress. My coding tasks where to implement Oauth using passport with google and facebook's APIs, as well as insure the front and back end coding for the event creation and invitation functionalities.
